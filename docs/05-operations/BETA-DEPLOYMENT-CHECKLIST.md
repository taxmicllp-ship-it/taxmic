# Beta Deployment Checklist

**Project:** Taxmic  
**Phase:** 7 — Beta Launch Preparation  
**Date:** 2026-03-17  
**Applies to:** Staging and Production deployments

---

## Pre-Deployment

- [ ] Confirm all database migrations are applied (`prisma migrate status` shows no pending migrations)
- [ ] Replace `JWT_SECRET` with a cryptographically strong secret (minimum 32 random bytes — do NOT use `"dev-secret-change-in-production"`)
- [ ] Set `NODE_ENV=production`
- [ ] Switch `STRIPE_SECRET_KEY` from test key (`sk_test_...`) to live key (`sk_live_...`)
- [ ] Switch `STRIPE_WEBHOOK_SECRET` from test secret (`whsec_...`) to the live webhook secret from the Stripe dashboard
- [ ] Confirm `DATABASE_URL` points to the correct staging/production database

---

## Database

- [ ] Run `prisma migrate deploy` to apply all pending migrations
- [ ] Verify the `roles` table contains at least the `owner` role:
  ```sql
  SELECT name FROM roles WHERE name = 'owner';
  ```
- [ ] If beta firms are not yet present, run the seed script:
  ```bash
  cd apps/api
  npx ts-node ../../scripts/seed-beta-firms.ts
  ```
- [ ] Confirm seed output shows 5 firms with status `created` (or `skipped` if already seeded)

---

## Application Startup

- [ ] Install API dependencies:
  ```bash
  cd apps/api
  npm install
  ```
- [ ] Install web dependencies:
  ```bash
  cd apps/web
  npm install
  ```
- [ ] Build the API:
  ```bash
  cd apps/api
  npm run build
  ```
- [ ] Build the web app:
  ```bash
  cd apps/web
  npm run build
  ```
- [ ] Start the API server:
  ```bash
  cd apps/api
  node dist/server.js
  ```
- [ ] Serve the web app build (via nginx, Caddy, or equivalent static host)

---

## Post-Deployment Verification

- [ ] Health endpoint responds with `{ "status": "ok" }`:
  ```bash
  curl https://<your-api-domain>/api/v1/health
  ```
- [ ] Login works for at least one beta firm credential:
  - Email: `admin@betafirm1.com`
  - Password: `BetaPass1!`
  - URL: `http://localhost:3001/login` (or your deployed frontend URL)
- [ ] Stripe webhook endpoint is reachable from the Stripe dashboard:
  - Register `POST https://<your-api-domain>/api/v1/payments/webhook` in the Stripe dashboard
  - Send a test event and confirm `200` response

## Known Limitations (Beta)

These are accepted limitations for the beta period and do not block launch:

| Limitation | Reason | Resolution |
|---|---|---|
| Email sending non-functional | `SES_FROM_EMAIL` not set in `.env` | Configure AWS SES and set `SES_FROM_EMAIL` before production |
| File storage is local disk | `STORAGE_PROVIDER` defaults to `local` | Switch to `s3` with AWS credentials for production |
| Error tracking inactive | `SENTRY_DSN` not set | Configure Sentry before production |
| No CORS origin restriction | `FRONTEND_URL` not validated by config | Add CORS origin check before production |

---

## Production-Only Requirements

These steps are **required before any production deployment** and are in addition to the checklist above:

- [ ] Replace `JWT_SECRET` with a value generated by:
  ```bash
  node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
  ```
- [ ] Set `SES_FROM_EMAIL` to a verified AWS SES sender address
- [ ] Switch `STORAGE_PROVIDER` to `s3` and set:
  - `AWS_REGION`
  - `AWS_ACCESS_KEY_ID`
  - `AWS_SECRET_ACCESS_KEY`
  - `AWS_S3_BUCKET`
- [ ] Set `SENTRY_DSN` for error tracking
- [ ] Set `FRONTEND_URL` and `API_URL` for CORS and redirect validation
- [ ] Rotate all Stripe keys to live keys
- [ ] Remove or restrict the `uploads/` local directory (not used in S3 mode)
